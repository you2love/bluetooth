蓝牙核心概念

蓝牙是一种工作在2.4GHz ISM频段的短距离无线通信协议，采用跳频扩频(FHSS)技术实现抗干扰，支持经典蓝牙(BR/EDR)和低功耗蓝牙(BLE)两种模式。

技术规格详解

📡 频段与信道

2.4GHz ISM频段 (2400-2483.5 MHz)

  • BLE: 40个信道 (每2MHz)
  • 经典蓝牙: 79个信道 (每1MHz)
  • 跳频: 1600hop/s (BLE) / 1600hop/s (经典)

📊 调制方式

GFSK (高斯频移键控)

  • BT=0.5, 调制指数=0.45-0.55
  • BLE 1M PHY: 1Mbps
  • BLE 2M PHY: 2Mbps
  • BLE Coded PHY: 500kbps/125kbps

🔋 功率等级

  • Class 1: 100mW (+20dBm), 100m
  • Class 2: 2.5mW (+4dBm), 10m
  • Class 3: 1mW (0dBm), 1m

🔒 调制差异

  • 经典蓝牙: BR/EDR, EDR(π/4-DQPSK, 8DPSK)
  • BLE: 单模设备, 无法与经典蓝牙兼容

双模 vs 单模

类型 支持模式 典型应用 芯片
双模 (BR/EDR+BLE) A2DP, HFP, SPP + GATT 手机、平板、PC Qualcomm, Broadcom
单模 (BLE Only) GATT only IoT设备、传感器 Nordic, TI, Dialog

PHY (Physical Layer)

  • RF射频收发
  • GFSK调制解调

  • Channel Index → RF频率

  • 状态机: Standby/Adv/Scann/Initiator/Connected
  • 广播/扫描/连接
  • 数据加解密
  • CRC校验

HCI (Host Controller Interface)

  • Command/Event/ACL/SCO
  • Flow Control
  • Buffer Management

L2CAP

  • Channel ID (CID) 复用
  • Segmentation/Reassembly
  • Flow Control
  • LE Credit Based Flow

配对模式

传统配对 (Legacy)

基于PIN码，安全性较低

  • 固定PIN码
  • 易受中间人攻击
  • 兼容旧设备

安全简单配对 (SSP)

基于椭圆曲线加密，安全可靠

  • 无需输入PIN码
  • 防中间人攻击
  • 多种验证方式

连接事件时序

// BLE 连接事件时序
// [T0] Connection Event Start
// [T0] Subevent #1: Data Exchange
// [T1] Subevent #2: (if Coded PHY)
// [Tx] ...
// [T_end] Connection Event End
//
// 多个连接事件之间空闲，可以省电
// 设备可选择不响应特定事件(slave latency)

// 传输窗口 (Transmit Window)

    安全威胁与防护

    ⚠️ 威胁：中间人攻击

    攻击者在配对过程中拦截通信

    ✅ 防护：使用SSP配对

    Numeric Comparison或Passkey Entry可防止MITM

    ⚠️ 威胁：重放攻击

    攻击者重放捕获的配对数据

    ✅ 防护：使用随机数

    配对过程使用加密随机数，防止重放

    ⚠️ 威胁：跟踪攻击

    通过广播地址跟踪设备

    ✅ 防护：使用随机地址

    定期更换随机广播地址，保护隐私

    ⚠️ 威胁：拒绝服务

    大量连接请求导致服务不可用

    MTU协商

    // MTU (Maximum Transmission Unit)
// 默认: 23 bytes (BLE 4.0)
// 最大: 512 bytes (BLE 5.0)

// ATT MTU = 协议开销 + 数据
// 23 = 4(ATT Header) + 20(L2CAP payload) - 1(MIC)
// 512 = 3(ATT Header) + 512(L2CAP)

// 协商流程
// 1. Client: Exchange MTU Request (client_rx_mtu)
// 2. Server: Exchange MTU Response (server_rx_mtu)
// 3. 实际MTU = min(client_rx_mtu, server_rx_mtu)

// 性能对比
// MTU=23: 每次传输20字节数据
// MTU=512: 每次传输512-3=509字节数据
// 大数据: 提升25倍吞吐量

    数据分片与重组

    // 当数据 > MTU-3 时需要分片

// 分包发送示例
def send_long_data(conn, handle, data, mtu=20):
    """大数据分包发送"""
    for i in range(0, len(data), mtu):
        chunk = data[i:i+mtu]
        conn.write_gatt_char(handle, chunk)
        await asyncio.sleep(0.01)  # 避免拥塞

        print(f"已连接: {client.is_connected}")

        # 获取所有服务
        for service in client.services:
            print(f"服务: {service.uuid}")
            for char in service.characteristics:
                print(f"  特征: {char.uuid}, 属性: {char.properties}")

        # 订阅通知
        def notification_handler(sender, data):
            print(f"收到通知: {data}")

        await client.start_notify(CHARACTERISTIC_UUID, notification_handler)

        # 读取数据
        data = await client.read_gatt_char(CHARACTERISTIC_UUID)
        print(f"读取数据: {data}")

        # 写入数据
        await client.write_gatt_char(CHARACTERISTIC_UUID, b"Hello BLE")

        await asyncio.sleep(10)  # 保持连接
        await client.stop_notify(CHARACTERISTIC_UUID)

asyncio.run(connect_and_notify())

package main

import (
    "context"
    "encoding/hex"
    "fmt"
    "time"

    "github.com/go-ble/ble"
    "github.com/go-ble/ble/examples/lib/gatt"
)

const (
    deviceAddr  = "AA:BB:CC:DD:EE:FF"
    serviceUUID = "180f"  // 电池服务
    charUUID    = "2a19"  // 电池电量特征
)

func main() {
    ctx := ble.WithSigHandler(context.WithTimeout(context.Background(), 10*time.Second))

    // 扫描设备
    cln, err := ble.Connect(ctx, deviceAddr)
    if err != nil {
        panic(err)
    }
    defer cln.CancelConnection()

    // 等待连接
    if !cln.Connected() {
        fmt.Println("连接超时")
        return
    }
    fmt.Println("已连接")

    // 发现服务
    svcs, err := cln.DiscoverServices(nil)
    if err != nil {
        panic(err)
    }

    for _, svc := range svcs {
        fmt.Printf("服务: %s\n", svc.UUID.String())
        chars, _ := cln.DiscoverCharacteristics(nil, svc)
        for _, char := range chars {
            fmt.Printf("  特征: %s\n", char.UUID.String())
        }
    }

    // 读取特征值
    char, err := cln.DiscoverCharacteristic(ble.MustParse(charUUID), nil)
    if err != nil {
        panic(err)
    }

    b, err := cln.Read(char)
    if err != nil {
        panic(err)
    }
    fmt.Printf("电池电量: %d%%\n", b[0])

    // 订阅通知
    ch := make(chan []byte, 5)
    if err := cln.Subscribe(char, false, func(req []byte) { ch <- req }); err != nil {
        panic(err)
    }

    // 写入数据
    data := []byte("Hello BLE")
    if err := cln.WriteCharacteristic(char, data, false); err != nil {
        panic(err)
    }

    // 保持连接
    select {}
}

            }
          });
        });
      }
    });

// 连接蓝牙设备
connectDevice() {
    const deviceId = this.data.targetDevice.deviceId;

    wx.createBLEConnection({
      deviceId,
      success: () => {
        console.log('连接成功');
        this.getBLEServices(deviceId);
      },
      fail: (err) => {
        console.error('连接失败', err);
      }
    });
},

// 获取服务列表
getBLEServices(deviceId) {
    wx.getBLEDeviceServices({
      deviceId,
      success: (res) => {
        const services = res.services;
        // 查找目标服务
        const targetService = services.find(s => s.isPrimary);
        if (targetService) {
          this.getBLECharacteristics(deviceId, targetService.uuid);
        }
      }
    });
},

// 获取特征值
getBLECharacteristics(deviceId, serviceId) {
    wx.getBLEDeviceCharacteristics({
      deviceId,
      serviceId,
      success: (res) => {
        const chars = res.characteristics;
        chars.forEach(char => {
          // 订阅通知
          if (char.properties.notify || char.properties.indicate) {
            wx.notifyBLECharacteristicValueChange({
              deviceId,
              serviceId,
              characteristicId: char.uuid,
              state: true,
            });
          }

          // 读取数据
          if (char.properties.read) {
            wx.readBLECharacteristicValue({
              deviceId,
              serviceId,
              characteristicId: char.uuid,
            });
          }
        });
      }
    });
},

// 监听特征值变化
onNotifyChange() {
    wx.onBLECharacteristicValueChange((characteristic) => {
      const buffer = characteristic.value;
      const data = new Uint8Array(buffer);
      console.log('收到数据:', data);
    });
},

// 写入数据
writeData() {
    const buffer = new ArrayBuffer(5)
    const dataView = new DataView(buffer)
    dataView.setUint8(0, 0x01)
    dataView.setUint8(1, 0x02)
    dataView.setUint8(2, 0x03)

    wx.writeBLECharacteristicValue({
      deviceId: this.data.deviceId,
      serviceId: this.data.serviceId,
      characteristicId: this.data.characteristicId,
      value: buffer,
      success: () => {
        console.log('写入成功');
      }
    });
},

// 断开连接
closeConnection() {
    wx.closeBLEConnection({
      deviceId: this.data.deviceId,
      success: () => {
        console.log('已断开');
      }
    });
}

    BLE开发核心

    GAP角色与模式

    Peripheral (周边)

    广播设备，被中心扫描连接

    • 典型: 传感器、手环
    • 功耗: 广播 > 待机
    • 单连接

    Central (中心)

    扫描并连接周边设备

    • 典型: 手机、PC
    • 多连接支持
    • 发起LL连接请求

    Observer (观察者)

    仅扫描，不连接

    • 典型: 蓝牙定位器

    Broadcaster (广播者)

    仅广播，不连接

    • 典型: Beacon
    • iBeacon/Eddystone

    广播类型与参数

    类型 PDU 用途 响应
    ADV_IND 可连接非定向 一般广播 SCAN_REQ / CONNECT_IND
    ADV_DIRECT_IND 可连接定向 快速重连 CONNECT_IND
    ADV_SCAN_IND 可扫描非定向 获取设备名称 SCAN_RSP
    ADV_NONCONN_IND 不可连接非定向 Beacon 无

    GATT服务设计

    服务声明

    • Handle: 0x0001
    • UUID: 0x2800 (Primary)
    • Value: Service UUID

    特征值声明

    • Handle: 0x0002
    • UUID: 0x2803
    • Properties: RW/N/Ind

    特征值

    • Handle: 0x0003
    • UUID: 自定义/标准
    • Value: 数据

    描述符 (可选)

    • CCC: 0x2902 (通知)
    • Format: 0x2901
    • User Desc: 0x2900

    ATT操作码

    Opcode 名称 方向 可靠性
    0x03 Read By Type Request C→P 需要
    0x04 Read By Type Response P→C 需要
    0x0B Write Request C→P 需要
    0x0C Write Response P→C 需要
    0x1B Handle Value Notification P→C 不需要
    0x1D Handle Value Indication P→C 需要

    常见问题 - 连接

    ❓ 设备无法被发现

    可能原因：

    • 设备未开启蓝牙或未进入可发现模式
    • 设备距离太远（超过10米）
    • 干扰严重（WiFi、微波炉等2.4GHz设备）
    • 设备名称设置错误

    解决方案：

    • 确保设备处于可发现模式
    • 缩短设备距离
    • 远离干扰源
    • 检查设备广播设置

    ❓ 连接经常断开

    可能原因：

    • 连接间隔设置不当
    • 信号强度不足（RSSI < -80dBm）
    • 设备进入低功耗模式
    • 超时时间设置过短

    解决方案：

    • 调整连接参数
    • 缩短设备距离
    • 检查从设备延迟设置
    • 增加超时时间

    ❓ 连接建立时间过长

    可能原因：

    • 广播间隔过大
    • 扫描间隔过大
    • 设备数量过多

    解决方案：

    • 减小广播间隔（推荐20-40ms）
    • 减小扫描间隔
    • 使用白名单过滤

    ❓ 多设备连接不稳定

    可能原因：

    • 连接总数超过硬件限制
    • 连接参数冲突
    • 带宽不足

    解决方案：

    • 检查硬件最大连接数
    • 优化连接参数
    • 减少数据传输频率

    常见问题 - 配对

    ❓ 配对失败

    可能原因：

    • IO能力不匹配
    • 用户取消配对
    • 配对超时
    • 设备不支持SSP

    解决方案：

    • 检查设备IO能力配置
    • 确保用户完成配对流程
    • 增加配对超时时间
    • 使用兼容的配对模式

    ❓ PIN码验证失败

    可能原因：

    • PIN码输入错误
    • PIN码不匹配
    • 设备固定PIN码

    解决方案：

    • 重新输入正确的PIN码
    • 确认设备PIN码
    • 使用SSP配对避免PIN码

    ❓ 配对后无法自动连接

    可能原因：

    • 设备未保存绑定信息
    • 设备白名单未添加
    • 广播设置改变

    解决方案：

    • 确保保存链路密钥
    • 将设备添加到白名单
    • 保持广播设置一致

    ❓ 每次都需要重新配对

    可能原因：

    • 设备不支持绑定
    • 存储空间不足
    • 固件问题

    解决方案：

    • 确认设备支持绑定
    • 清理存储空间
    • 更新设备固件

    常见问题 - 音频

    ❓ 音频卡顿或延迟

    可能原因：

    • 信号干扰严重
    • 连接间隔过大
    • 编解码器参数不当
    • 设备性能不足

    解决方案：

    • 减少干扰源
    • 调整连接参数（降低间隔）
    • 使用低延迟编解码器
    • 升级设备硬件

    ❓ 音质不佳

    可能原因：

    • 使用低码率编解码器
    • 信号强度不足
    • 编解码器不匹配

    解决方案：

    • 使用高质量编解码器（aptX/LDAC）
    • 缩短设备距离
    • 确认双方支持的编解码器

    ❓ 无法使用高质量编解码器

    可能原因：

    • 设备不支持该编解码器
    • 编码器未启用
    • 协议版本限制

    解决方案：

    • 检查设备兼容性
    • 在设置中启用编码器
    • 更新蓝牙版本

    ❓ 音频中断后无法恢复

    可能原因：

    • A2DP流断开
    • 设备状态异常
    • 连接参数改变

    解决方案：

    • 重新启动音频流
    • 断开并重新连接
    • 恢复连接参数

    解决方案：

    • 确保进入低功耗模式
    • 使用间歇广播

    可能原因：

    • 电池容量小
    • 使用场景不合理

    解决方案：

    • 使用更大容量电池
    • 优化使用场景
    • 优化固件代码

    可能原因：

    • 设备不支持BAS服务
